# ADR 0003: no agregar backend al MVP

- Estado: aceptado.
- Contexto: mapa, misiones y progreso inicial no necesitan coordinación entre usuarios.
- Decisión: usar archivos estáticos y persistencia local del navegador.
- Alternativas: una API con base de datos añade operación, secretos y fallos sin aportar al MVP.
- Consecuencias: no habrá sincronización entre dispositivos ni ranking global en esta etapa.
